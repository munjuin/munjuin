## 🚀 Projects Highlight
*아래 Pinned Repositories 프로젝트들의 **개발 의도(Why & How)**입니다.*

### 1. ts-lodashtype-definition
**"타입 시스템의 한계를 시험하고 정교한 설계 능력 기르기"**
- **문제 의식:** 단순히 타입을 '지정'하는 것을 넘어, 복잡한 유틸리티 라이브러리가 어떻게 전 세계 개발자에게 완벽한 추론을 제공하는지 그 원리가 궁금했습니다.
- **해결 접근:** - Lodash의 핵심 함수들을 선정하여 `Generics`, `Conditional Types`, `Mapped Types`를 이용해 직접 타입 정의.
    - 고차 함수와 객체 조작 시 발생할 수 있는 타입 단절 문제를 해결하며 **정적 분석의 안정성** 극대화.

### 2. Jewelry Shop Project (진행 중)
**"라이브러리 없이 순수 Node.js 아키텍처 구축하기"**
- **문제 의식:** 프레임워크가 자동화해 주는 기능들을 직접 구현하며 `MVC 패턴`과 `DB 설계`의 중요성을 체감하고 싶었습니다.
- **해결 접근:**
    - `JSON` 타입을 활용하여 반지 호수/색상 등 복잡한 옵션 데이터를 유연하게 설계.
    - 현재 JS 기반의 MVP를 **TypeScript로 마이그레이션**하여 대규모 데이터 처리의 신뢰성 확보 중.

### 3. create-express-esm (NPM Package)
**"내가 불편해서 직접 만든 개발 도구"**
- **문제 의식:** 기존 Express 생성기가 CJS 기반이라, 최신 ESM 문법을 적용하려면 매번 초기 설정에 시간을 낭비했습니다.
- **해결 접근:** `npm init` 한 번으로 최신 스택(ESM)이 적용된 보일러플레이트를 생성하는 **CLI 도구 직접 개발 및 배포**.
